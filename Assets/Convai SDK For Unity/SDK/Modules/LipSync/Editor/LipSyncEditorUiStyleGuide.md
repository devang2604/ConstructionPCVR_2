# LipSync Editor UI Style Guide

## Scope

This guide defines the IMGUI design system used by LipSync editor surfaces in the Convai SDK package.

## Core Rules

- Use `ConvaiLipSyncEditorThemeTokens` for colors and spacing tokens.
- Use `ConvaiLipSyncSectionChrome` for all collapsible section headers and bodies.
- Persist section state with `ConvaiLipSyncSectionStateStore`.
- Resolve branding textures via `ConvaiLipSyncIconProvider` only.
- Do not allocate `GUIStyle` in per-frame draw paths. Cache in `InitializeStyles`.

## Standard Header Layout

- Order: `icon -> chevron -> title`
- Entire row is clickable for expand/collapse.
- Divider line is shown under expanded headers.
- Dark theme is the premium target.

## State Keys

Section keys are generated by:

`Convai.LipSync.<EditorType>.<SectionId>.Expanded`

Examples:

- `Convai.LipSync.ComponentEditor.CoreSetup.Expanded`
- `Convai.LipSync.MapAssetEditor.Mappings.Expanded`

## How To Add A New Section

1. Define a stable section id constant in the host editor.
2. Load default state in `OnEnable`:
   `ConvaiLipSyncSectionStateStore.Get(hostId, sectionId, defaultValue)`.
3. Save state in `OnDisable`:
   `ConvaiLipSyncSectionStateStore.Set(hostId, sectionId, value)`.
4. Draw header using `ConvaiLipSyncSectionChrome.DrawHeader`.
5. Draw body using:
   - `ConvaiLipSyncSectionChrome.BeginBody(...)`
   - content
   - `ConvaiLipSyncSectionChrome.EndBody()`

## Review Checklist

- No hardcoded editor icon file paths.
- No `EditorPrefs` ad-hoc keys for section state.
- No duplicated section header implementations.
- No per-frame style allocations in section chrome paths.
